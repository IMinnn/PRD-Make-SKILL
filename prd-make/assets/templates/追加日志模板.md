# 追加日志

## 使用说明

- 每次追加、修订、确认、跳过或回滚 PRD 内容后，都要记录一条日志。
- “模块/章节”填写本次影响的 PRD 正文单元、功能模块、模块规划、待决策事项或其他工作区文件。
- “动作”可填写初始化、追加、修订、确认、待确认、同步规划、关闭待决策事项等。
- “评审状态”用于标记当前内容是否已由用户确认，例如 `待确认`、`已确认`、`需调整`、`自动继续`。
- “备注”记录本次变更原因、用户确认依据、影响范围或下一步。
- 该章节仅指导你如何去生成日志文档，在正式生成时不要输出本章节内容。
